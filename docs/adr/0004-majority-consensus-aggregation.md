# ADR-004: 综合替身多数共识聚合

**日期**: 2026-06-10  
**状态**: 已采纳  
**决策者**: 用户  

---

## 背景

用户在不同好友面前有多个替身（"张三眼中的我"、"李四眼中的我"）。综合替身是这些替身的聚合，表示用户的共性特征，作为每个替身的底层兜底数据。

聚合规则的选择影响综合替身的准确性和覆盖面。

## 方案对比

### 方案 A：严格交集
只保留所有替身共有的标签。示例：只有"内向"在所有替身中都出现时才保留。  
缺点：过于苛刻，"爱吃辣"只在一个视角中出现不代表不是共性。

### 方案 B：多数共识 ✓
保留在多数替身（>50%）中出现的标签。  
优点：平衡了准确性和覆盖面。

### 方案 C：LLM 自由判断
全部标签喂给 LLM，由 LLM 自行提炼共性。  
缺点：主观性太强，不可复现。

## 决策

**采纳方案 B：多数共识。**

- OCEAN 分数：所有人格替身的算术平均值
- 风格标签：交由 LLM 做去重合并，但保留规则是"多数替身共有的标签"
- 偏好/事实标签：同样多数共识

## 影响

- 综合替身不会因为一个极端替身而被带偏
- 首次只有 1 个替身时，综合替身 = 该替身（降级逻辑）
- 替身数量越多，综合替身越准确
- 聚合触发时机：每次新导入聊天记录生成新替身后触发重聚合
