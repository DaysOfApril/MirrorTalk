# MirrorTalk — 领域术语表

本文档定义 MirrorTalk 的领域概念（Glossary），不包含任何实现细节。

---

## 核心实体

### 虚拟好友 (Virtual Friend)
基于真实好友的微信聊天记录构建的 AI Agent。具备该好友的说话风格、口头禅、已知事实。用户可直接在 MirrorTalk 中与之对话，体验"和真人好友聊天"的感觉。

### 用户替身 (User Persona)
基于用户在某特定好友面前的发言构建的 AI Agent。替身模拟"用户在该好友眼中是什么样的人"，用于替用户生成回复。每个好友对应一个独立的用户替身。

### 综合替身 (Aggregated Persona)
将所有用户替身按多数共识聚合后形成的"用户共性画像"。作为各替身的底层兜底数据。

### 人格 (Persona)
对虚拟好友或用户替身的语言风格、性格特征、偏好、身份的结构化描述。包含两个层面：
- **风格标签**：口头禅、句式、表情包习惯、语气（可直接指导回复生成）
- **OCEAN 分数**：五维度人格量化值（用于跨替身对比与聚合）

### 知识库 (Knowledge Base)
跨对话持久化的事实条目集合。来源包括：聊天记录中提取的原子事实、外部文件（txt/md）导入的知识点。知识库条目按来源区分访问权限。

### 原子事实 (Atomic Fact)
从聊天记录或外部文件中经 LLM 抽取的最细粒度事实条目。不可再分。例如："用户喜欢吃辣"、"张三讨厌香菜"。原子事实是知识库的基本存储单元，也是画像聚合的输入数据。

### 记忆 (Memory)
知识库中条目的统称。Agent 通过 `recall` 工具按需检索记忆。记忆附带置信度（confidence）和重要性（importance）评分。

### 对话会话 (Conversation)
用户与虚拟好友（或替身回复页面）之间的一次多轮对话。每次新建对话默认从空白开始，用户可手动恢复历史会话继续对话。

### Provider
大模型服务提供商的抽象。内置预设（OpenAI、DeepSeek、Qwen 等）和自定义入口（用户自填 BaseURL + 协议 + 模型）。Embedding 和 Rerank 同样遵循 Provider 模式。

---

## 数据边界

| 数据 | 描述 | 访问权限 |
|------|------|:--:|
| 外部文件 | 用户导入的 txt/md 文档 | 所有 Agent 可读 |
| 用户发言事实 | 从用户侧消息中提取 | 仅用户替身 Agent 可读 |
| 好友发言事实 | 从好友侧消息中提取 | 所有 Agent 可读 |
| 共同事实 | 双方对话中的共享信息 | 所有 Agent 可读 |

---

## 检索模式

### 工具式检索 (Tool-based Recall)
记忆/知识库不在对话开始时全量注入。Agent 在对话过程中自主决定何时调用 `recall` 工具查询。模拟"聊到才想起来"的人脑自然模式。

### 混合检索 (Hybrid Retrieval)
单次 `recall` 调用内部自动执行：关键词全文搜索（FTS） + 向量语义搜索 → RRF 排名融合 → Rerank 精排。Agent 不需要感知底层检索策略。
