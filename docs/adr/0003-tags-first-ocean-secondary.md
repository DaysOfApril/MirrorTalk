# ADR-003: 画像体系：标签为主、OCEAN 为辅

**日期**: 2026-06-10  
**状态**: 已采纳  
**决策者**: 用户  

---

## 背景

用户替身需要结构化画像指导回复生成。两种可能的画像体系：
- **OCEAN（大五人格）**：五个维度连续值，心理学标准模型
- **自定义标签**：接地气的自然语言描述

## 方案对比

### OCEAN 标准模型
`{openness: 0.65, extraversion: 0.3, ...}`

优势：可数值对比、学术规范感强。  
劣势：从聊天记录推 OCEAN 分数极不可靠；Agent 拿到数值无法生成具体风格的回复。

### 自定义标签
`{性格: ["内向", "细心"], 口头禅: ["笑死"], 句式: "短句为主"}`

优势：LLM 直接理解、直接指导回复生成、抽取精度高。  
劣势：难以做跨替身量化对比。

### 混合方案 ✓
标签为主（注入 system prompt 指导回复），OCEAN 为辅（仅用于跨替身聚合时的加权对比）。

## 决策

**采纳混合方案：标签为主、OCEAN 为辅。**

## 影响

- 风格标签直接注入 Agent 的 system prompt，指导回复生成
- OCEAN 分数仅用于综合替身的聚合计算，不注入对话上下文
- 画像 Pipeline 需要两次独立的 LLM 抽取：事实抽取 + 风格抽取
- 面试展示时，标签体系更直观、更易于解释
